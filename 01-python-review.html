NumPy Arrays — the super-fast row of boxes

A NumPy array is just a row (or grid) of numbers stored in identical boxes. Because every box is the same size, the computer can blast through millions of them at once — far faster than a regular Python list.

🥚
Real-life analogy
Think of an egg carton. Every slot is identical, every egg sits in its place, and you can grab egg #5 instantly — you don't have to count from the front. That's a 1-D NumPy array.
Step 1 of 4
An array is a row of identically-sized boxes. Each holds one number.
# In code, this whole story is just two lines: import numpy as np arr = np.array([10, 20, 30, 40, 50, 60]) arr * 2 # → array([20, 40, 60, 80, 100, 120])
2

Pandas DataFrame — the smart spreadsheet

A DataFrame is a labelled table: rows are records, columns are properties. It's a spreadsheet with super-powers — every column knows its own name and type.

📒
Real-life analogy
Imagine a school attendance register. Down the side are students (rows), across the top are columns like Name, Age, City, Marks. That's a DataFrame.
Step 1 of 5
We start with an empty table. Watch the structure build piece by piece.
import pandas as pd df = pd.DataFrame({ "name": ["Asha", "Ben", "Cara", "Dev"], "age": [23, 31, 28, 45], "city": ["Pune", "Pune", "Delhi", "Delhi"], "marks": [82, 76, 91, 65] })
3

Filtering — keep only the rows you want

A filter asks every row a yes/no question. Rows that answer "yes" stay, rows that answer "no" are dropped. The original data is untouched — you get a smaller copy.

🪜
Real-life analogy
Think of a sieve over flour. The big lumps stay on top, the fine flour falls through. Filtering says: "let through every row where marks > 75."
Step 1 of 5
Same table as before. We will ask each row: is your marks bigger than 75?
# Boolean mask: True / False for every row mask = df["marks"] > 75 top_scorers = df[mask] # keep only the True rows # short form: df[df["marks"] > 75]
4

GroupBy — split, summarise, combine

GroupBy says: "bucket the rows by some column, then run one calculation per bucket." It's how you turn a big table into a tidy summary.

🧺
Real-life analogy
Picture laundry day. You sort socks into piles — whites, colours, darks — and then count each pile. Group by colour, then count. That's exactly groupby.
Step 1 of 4
The same 4 students. We want average marks per city. Watch them split and re-form.
df.groupby("city")["marks"].mean() # city # Delhi 78.0 # Pune 79.0
5

Merging — joining two tables by a common key

When two tables share a column (an ID, a name, a date), merging zips them together row-by-row. It's how isolated bits of information become a complete picture.

🧩
Real-life analogy
A movie ticket and a popcorn coupon in your wallet. Both have your booking ID. Bring them together by that ID and now you have the full snack-and-show combo.
Step 1 of 4
Two separate tables — one with names, one with scores. They share a column: id.
pd.merge(students, scores, on="id") # pairs each row by matching id — full picture in one table
6

Pivot Tables — reshape rows into a grid

A pivot turns a long list into a cross-tab: one variable becomes the rows, another becomes the columns, and a number fills each cell. It's a snapshot of relationships at a glance.

📅
Real-life analogy
A weekly class timetable: days across the top, periods down the side, the subject in each cell. From a messy diary, you get an instant grid you can read in seconds.
Step 1 of 4
A sales log — every transaction is one row. Hard to spot patterns when it's this long.
df.pivot_table( index="city", columns="product", values="sales", aggfunc="sum" )
